## What Went Well

### Mock Mode as a Strategic Unlocker
`NEXT_PUBLIC_MOCK_WORLDID=true` was the single most important architectural decision of the entire sprint. It allowed every team member and every downstream story (Epics 2–5) to develop without a live World ID simulator. The dual-path design (mock vs real) was clean and the single env-var toggle never caused confusion.

### Scaffolding-First Strategy
Stories 1.1–1.3 benefited from a codebase that already had significant scaffolding in place. The dev agent pattern of "validate, harden, fill gaps — don't rewrite" was well-executed and avoided redundant work.

### Review Caught Critical Security Issues Early
Story 1.5's review flagged an unauthenticated admin reset endpoint (publicProcedure with no key validation) before it shipped. Without the code review step, this would have been a demo-breaking security hole.

### Test Coverage Grew Progressively
43 tests at story 1.1, growing to 132 by story 1.4. Each story added meaningful tests rather than skipping them under hackathon pressure.

---

## What Was Difficult

### Non-Deterministic Mock Nullifier (Story 1.1)
The initial `Date.now()` mock nullifier caused flaky tests and unpredictable behavior across sessions. Switching to a hash-based deterministic nullifier fixed it but consumed investigation time that wasn't planned.

### Incomplete Review Items in 1.4
Story 1.4's review findings (hardcoded stats in header, incomplete compact badge, unsafe string truncation) were left unchecked (`[ ]` not `[x]`). The story was merged with known bugs. For a hackathon this is acceptable, but the deferred-work.md should track these explicitly.

### tRPC Cookie Inconsistency
Story 1.1 uncovered that `auth.register` didn't set the httpOnly cookie while `/api/verify-proof` did. This dual-path inconsistency required later investigation in story 1.3 to resolve cleanly.

---

## Lessons Learned

1. **Document mock vs real divergence explicitly.** Any place where mock mode differs from production behavior should have a `// MOCK:` comment. Devs in later epics repeatedly had to re-discover what mock mode skipped.

2. **Mark review items resolved or deferred — never leave them in a mixed state.** Story 1.4 has unchecked `[ ]` review findings. Either fix them, mark them `[x] [Defer]`, or create a tracking item in deferred-work.md.

3. **Validate session consistency (cookie vs token) before downstream stories consume it.** The tRPC/REST cookie split caused confusion across stories 2–4.
