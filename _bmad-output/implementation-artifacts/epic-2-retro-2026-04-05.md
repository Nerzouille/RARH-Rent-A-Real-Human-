# Epic 2 Retrospective — Autonomous Agent Integration (World AgentKit)

**Date:** 2026-04-05
**Epic:** 2 — Autonomous Agent Integration
**Stories:** 2.1 · 2.2 · 2.3 · 2.4 · 2.5
**Status at retro:** All 5 stories done

---

## What Went Well

### Fast Pivot on SDK Incompatibility
`@worldcoin/agentkit` was incompatible with Next.js 16. Instead of blocking, the team immediately moved to a robust regex-based header validation (`/^AgentKit (0x[0-9a-fA-F]{40})$/i`). The fallback was documented in the story spec upfront as an explicit contingency — this forward-thinking prevented a costly blocker.

### Fail-Soft AgentBook Lookup
The `agentBookVerified: false` fallback for unreachable AgentBook is the right call for a demo. It means the platform never hard-fails due to an external dependency. This pattern should be the default for all external integrations.

### Central Schema for Auth Regex
After review caught a duplicated regex in story 2.1, `AGENTKIT_HEADER_RE` was moved to `src/lib/schemas/index.ts` as the single source of truth. This is the correct architectural move and was applied consistently in subsequent stories.

### MCP Tool Surface is Clean
The 5 exposed tools (`get_identity`, `list_tasks`, `create_task`, `get_task_status`, `validate_task`) were well-scoped. No tool does too much; boundaries are clear.

---

## What Was Difficult

### SDK Beta Quality
The AgentKit SDK targets Hono middleware and isn't built for Next.js App Router. This is a pre-production reality of building on bleeding-edge hackathon tracks. The regex fallback works for demo purposes but is not production-grade (no signature verification).

### AgentBook Lookup Coverage Gap
`lookupAgentBookOwner()` had no direct test coverage until story 2.2, and even then MCP tool-level tests were explicitly deferred. The MCP route is the most untested surface in the codebase.

### Header Format Discovered Late (Story 5.2)
A critical bug — invalid `x-agentkit-auth` header format in the judges dashboard (story 5.2) — wasn't caught until a later epic. This suggests the MCP auth path needs an integration test that actually fires the full request.

---

## Lessons Learned

1. **Always define SDK fallback strategies before starting the story.** Story 2.1 had this in the spec — it saved significant time. Apply this pattern to any external SDK that's in beta.

2. **The MCP route is the highest-risk untested path.** All 5 MCP tools, the `withAgentAuth()` middleware, and AgentBook lookup are exercised only manually. A single integration test firing a real JSON-RPC request against `/api/mcp` would have caught the story 5.2 header bug days earlier.

3. **Fail-soft is correct for external dependencies, but document the degraded state.** When `agentBookVerified: false`, the UI should make this visible to the user, not silently drop the information.
